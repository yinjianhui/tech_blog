**一、核心架构**

**二、具体实现**

#### 1. **数据存储层**（文件系统）

**为什么选文件而不是数据库？**
- 简单、可靠、易于备份
- OpenClaw workspace 就是 git 仓库，天然版本控制
- JSON 格式易于人类阅读和机器解析

#### 2. **定时任务层**（OpenClaw Cron）

**设置 5 个 cron job**，每个对应一个提醒时间：

```javascript
// 每个 job 的 payload 示例
{
  "kind": "cron",
  "expr": "20 8 * * *",        // 每天 08:20
  "tz": "Asia/Shanghai",
  "payload": {
    "kind": "systemEvent",
    "text": "CHECK_TODO:8:20"
  }
}
```

**工作流程**：
1. cron 触发 → 发送 systemEvent 到当前 session
2. 我收到消息 → 读取当日待办文件
3. 格式化待办清单 → 发消息给你
4. 等待回复 → 解析反馈 → 更新 JSON 文件

#### 3. **对话交互层**（消息监听）

**两种触发模式**：

**模式 A：被动监听**（每次对话时检查）
```
辉哥发消息 → 我检查是否包含待办关键词
           ↓
       是 → 提取内容 → 写入文件 → 确认
       否 → 正常对话
```

**模式 B：主动提醒**（cron 触发）
```
cron 到点 → 发送 "CHECK_TODO:8:20"
          ↓
      读取今日待办 → 格式化 → 发送清单
          ↓
      等待用户回复 → 解析状态 → 更新文件
```

#### 4. **状态管理逻辑**

**解析用户反馈**（自然语言 → 状态更新）：
```javascript
function parseStatusUpdate(message, todos) {
  // 关键词匹配
  if (message.includes('完成了') || message.includes('搞定了')) {
    return 'completed';
  }
  if (message.includes('不需要') || message.includes('取消')) {
    return 'cancelled';
  }
  if (message.includes('进行中') || message.includes('正在做')) {
    return 'in_progress';
  }
  return 'pending';  // 默认
}
```

#### 5. **工作日记生成**

**每日 23:59 cron job**：
```javascript
{
  "kind": "cron",
  "expr": "59 23 * * *",
  "payload": {
    "kind": "systemEvent",
    "text": "GENERATE_WORK_LOG"
  }
}
```

**生成逻辑**：
```javascript
function generateWorkLog() {
  const file = `/root/.openclaw/workspace/memory/todo/daily/${today()}.json`;
  const todos = JSON.parse(readFile(file) || '[]');

  const completed = todos.filter(t => t.status === 'completed');
  const pending = todos.filter(t => t.status !== 'completed' && t.status !== 'cancelled');

  const log = `# ${today()} 工作日记

## ✅ 已完成 (${completed.length})
${completed.map(t => `- ${t.content}`).join('\n')}

## ⏳ 未完成 (${pending.length})
${pending.map(t => `- ${t.content} (${t.status})`).join('\n')}

---
生成时间：${new Date().toLocaleString('zh-CN')}
`;

  // 写入 diary 仓库
  const diaryPath = '/root/.openclaw/workspace/diary/工作日记';
  writeFile(`${diaryPath}/${today()}.md`, log);

  // Git push
  exec(`cd ${diaryPath} && git add . && git commit -m "工作日记: ${today()}" && git push`);
}
```

// storage.js - 数据存储层（示例）
function addTodo(content, level = 'daily', options = {}) {
  const todos = readTodos(level);

  const newTodo = {
    id: generateId(),
    content: content,
    level: level,
    status: options.status || 'pending',
