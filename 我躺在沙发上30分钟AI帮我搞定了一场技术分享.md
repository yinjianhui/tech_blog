## 01 晚上躺在沙发上，突然想起要做技术分享

时间：晚上 7:47

我正躺在沙发上刷手机，突然想起这周要做个技术分享，主题是 AI Coding。

问题来了：
- 内容完全没准备
- 团队大部分人只用过 ChatGPT/DeepSeek 的网页版
- 要讲进阶内容：Prompt、Skill、Agent
- 还要演示 Claude Code 的使用

按以前的套路，我得：
1. 爬起来开电脑
2. 搜索各种资料
3. 整理大纲
4. 写文档
5. 制作 PPT

起码几个小时。

但现在不一样了，我有 OpenClaw。

---

## 02 什么是 OpenClaw？

简单说，**OpenClaw 就是一个能帮你干活的 AI 助手**。

和 ChatGPT、DeepSeek 这些聊天机器人不一样，OpenClaw 能：
- 读写你电脑上的文件
- 执行命令（比如 git 操作）
- 搜索网络资料
- 持续对话，理解上下文
- 主动思考，不是被动回答

**你可以把它当成一个能真正帮你干活的技术伙伴。**

好了，进入正题。

---

## 03 7:47 - 一句话启动任务

我打开 OpenClaw（在飞书里），发了第一句话：

> "我是一名软件开发部门的员工，我探索了很多 AI 的用法。我们团队很多人还没有用过 AI，我想做一个技术分享。嗯，大纲技术分享的内容，我还没有写好，我咱们一起探讨这个要分享的内容。接下来你跟我探讨，不明白的地方你就向我询问，直到我们的分享内容完全确定下来。"

就这么一句话，OpenClaw 立刻开始工作。

---

## 04 7:47 - 8:00 - 确定分享方向

OpenClaw 并没有直接给我一个模板，而是开始**主动提问**：

> "好，这事儿有意思！技术分享 AI 用法，帮团队入门，很实用。先确认几个基础信息：
>
> 1. 受众水平：团队同事的技术背景大概是什么？资深开发、初级开发、还是混合？
> 2. 分享时长：预计多长时间？30分钟、1小时、还是更久？
> 3. 分享场景：是正式的会议室分享，还是轻松的 lunch & learn 风格？
> 4. 侧重点：你更想分享什么方向？"

你看，它不是瞎猜，而是**主动引导我思考**。

我继续回答：

> "我们团队的都是软件开发开发工程师有一定的开发基础，我的分享大概在 45 分钟左右；分享的风格大概采用就是说讲解原理加实操例子举例；分享的主题我想侧重于AI Coding，比如说，AI Coding 的工具，Claudecode 的使用为主；要向他们讲清楚什么是智能体？什么是 skill，如何在 Claude 中自定义 skill，自定义智能体，以及这些核心概念的区别，为什么要这么做，和 prompt 有什么区别和联系..."

OpenClaw 马上理解了，开始给出初步的时间分配建议。

---

## 05 8:00 - 8:15 - 核心概念讲解

接下来是最关键的部分：**讲清楚 Prompt、Skill、Agent 的区别**。

OpenClaw 问我：

> "关于 Agent 的讲解，有参考文章吗？"

我把我之前整理的资料发给它（一段关于 AI Agent 的详细说明），OpenClaw 立刻理解了，开始重新组织内容。

**关键对话来了**：

> 我："第二部分，核心概念那个地方，我希望每一个概念是一个单独的小节，比如说 prompt、skill、agent 都是单独的，然后再来一个小节是对比分析，然后把渐进式披露融合到 skill 中去，而不是单独说，在讲到 Agent 的时候，嗯，把我给你提供 Agent 的内容放在这个小节，懂了吗？对。"

OpenClaw 立刻调整了结构：
- 2.1 什么是上下文？
- 2.2 Prompt（简化，大家都懂）
- 2.3 Skill（包含渐进式披露概念）
- 2.4 Agent（完整讲解）
- 2.5 三个概念的对比分析

**太赞了，它完全理解了我的意思。**

---

## 06 8:15 - 8:17 - 调整结构

我又提出：

> "第三部分和第四部分有重复"

OpenClaw 马上发现：
- 第三部分讲了 CLAUDE.md、/compact、/clear
- 第四部分也讲了这些

它立刻重新整合，去掉了重复内容。

然后我说：

> "Prompt 可以简单说，因为大家都知道"

于是 OpenClaw 把 Prompt 部分简化到只有几行，重点放在 Skill 和 Agent 上。

## 11 OpenClaw 的核心能力

通过这次实战，我真正体会到了 OpenClaw 的强大：

### 1. 主动思考，不是被动回答
它不会给你一个模板就完事，而是**主动提问**，帮你理清思路。

### 2. 理解上下文
我们的对话持续了一个多小时，但它**始终记得前面的内容**，不会跑偏。

### 3. 精准执行指令
- "删除变化 4，注意别删多了" → 精准删除，不影响其他部分
- "同步更新大纲" → 自动找到对应文件并更新
- "推送到 GitHub" → 完整的 Git 工作流

### 4. 工具调用
- 读写文件
- Git 操作（add、commit、push）
- 搜索资料（虽然有些链接没打开，但展示了能力）

### 5. 持续迭代
根据我的反馈，它不断调整、优化，直到我满意。
